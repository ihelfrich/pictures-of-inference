# Process

How we write this book. Living document. When we change how we work, we update this.

## What this project is

Three things, kept distinct:

1. **The book.** *Pictures of Inference*. Web-first, print fallback. This repo.
2. **The figure and interactive layer.** Reusable assets: widgets, embeds, deck material. Lives in this repo, exported on demand.
3. **The empire.** Brand, courses, tutoring, community. Separate property. Links to the book; the book does not depend on it.

Conflating them produces drag. The book is the book.

## Format

Quarto. Source authoring in `.qmd`. Single source produces:

- HTML book (primary reading experience)
- PDF (offline and print fallback)

Why Quarto: math via KaTeX in HTML and LaTeX in PDF, code in any language, Observable JS for interactive figures, citations, cross-references, search, dark mode. Same stack as Wickham, Hyndman, and Çetinkaya-Rundel. The footprint of solved problems is large.

PDF preamble: `prose/_quarto-preamble.tex`. Contains custom commands, callout boxes, math macros. It does **not** contain `\documentclass` or geometry, because Quarto manages those.

Legacy LaTeX path: `prose/preamble.tex`, `prose/main.tex`. Kept for now as a fallback. Removed when Quarto is the only build anyone uses.

## Graphics

Three figure types, three pipelines.

**Static analytic figures.** matplotlib plus the `@poi_style` decorator. Already in the repo. Output: PDF for print, SVG for web. Source of truth: `pictures/figures/chNN.py`.

**Diagrams** (the data rectangle, vector projection, DAGs, the satellite swath). Authored in Python with vector primitives that extend `poi.style`. Same dual output. TikZ rejected because non-LaTeX humans cannot edit it.

**Interactive figures.** Observable Plot inside Quarto OJS chunks. Static snapshot for PDF. Used sparingly: only when interaction teaches something a static figure cannot.

We do not build a graphics engine. We build a thin, opinionated style library on top of existing tools.

## Datasets

Loaded by one function call: `from poi.data import trade`. That requires:

- `data/<slot>/raw/` — raw inputs, gitignored if large
- `data/<slot>/process.py` or `.R` — produces clean output
- `data/<slot>/clean/` — versioned, committed
- `data/<slot>/codebook.md` — variable definitions
- `data/<slot>/README.md` — sources and provenance
- `pictures/lib/poi/data.py` — loader functions

`data/trade/` currently holds a synthetic placeholder generated by `process.py`. Swapped for the real `EffDist_V2026` slice (or BACI / COMTRADE) before any chapter ships beyond drafts.

## Writing loop

Section, not chapter.

A section is 300 to 800 words. A chapter is 8 to 15 pages. Iterating on full chapters lets voice drift before the second half is read; iterating on sections keeps calibration tight.

Per chapter:

1. Agree on a one-page chapter spec: question, ideas, figures, dataset(s), running examples. No prose yet.
2. Draft section 1. Read, cut, redirect. Iterate until it sings.
3. Sections 2 through N, faster as voice calibrates for this chapter.
4. Stitch. Read aloud. Where it does not flow, fix.
5. Figures lock. Cross-references resolve.
6. Ship to a branch. Read on a different day. Last-mile edits.

Ian writes at least one paragraph of every chapter. Even badly. Gives a fixed point to triangulate against and keeps the book honest as his.

## Voice

`docs/voice/charter.md` is the contract. `docs/voice/targets.md` is the triangulation. When prose drifts, point at the charter, not at vibes. Voice samples from Ian and Liz land in `docs/voice/` as `ian-NNN.md` and `liz-NNN.md` and sharpen the charter further.

## Build

- `quarto preview` — live HTML rebuild during writing
- `quarto render` — full HTML and PDF build
- `make figures` — regenerate matplotlib figures
- `make check` — style check (banned phrases)

Target: under 30s for a full book build. If it grows past that, we fix it before continuing.

## Iteration unit and pace

Section per back-and-forth. Two to four sections per session is normal. A whole chapter in one sitting is rare and usually wrong.

The horizon is multi-year. Part One drafted by end of 2026 is a real target. The rest follows.

## Empire layer

Out of scope for this repo. Two decisions made now that affect downstream empire work:

**Licensing.** Prose: CC-BY-SA 4.0. Code and figures: MIT. Datasets: per-dataset, documented. The book is evangelistic; the empire monetizes on tutoring, courses, community, and embedded widgets, not on restricting access to the book.

**Domain.** Three properties:

- `ianhelfrich.com` — personal site, unchanged.
- The book — reserve a domain (`picturesofinference.org` or similar). Until then, GitHub Pages.
- Future empire domain — not yet, but reserve a name when one feels right.
